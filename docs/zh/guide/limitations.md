# 局限性

GDSQL 专为游戏配置数据设计，不是通用数据库。了解其局限性有助于合理使用。

## 性能

**纯 GDScript 实现。** 不适用于：每秒数千次查询、大规模数据分析、实时多人游戏状态同步。这类场景请考虑 SQLite（GDExtension）或服务器数据库。

## 数据规模

针对**数百到数万条记录**优化。超出范围时：查询执行时间明显增加、内存占用增长、文件 I/O 成为瓶颈。不适合数据仓库或数十万行以上的数据集。

## 并发

**不支持并发写入。** 所有数据库操作必须在**主线程**执行。如需后台处理：在主线程查询，缓存结果供后台线程只读使用。

## 无服务器模式

纯本地、基于文件的数据存储：无网络访问、无客户端-服务器架构、无多用户并发。

## 无参照完整性

外键关系仅在**应用层**存在，存储引擎不强制约束。

## 索引有限

索引在内存中维护，无持久化索引结构。

## 无 DDL 语句

结构变更（创建表、修改列）仅通过**可视化界面**进行。不支持 `CREATE TABLE`、`ALTER TABLE`、`DROP TABLE`。
