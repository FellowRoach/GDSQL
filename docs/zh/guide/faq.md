# 常见问题

## 基础问题

### GDSQL 是真正的数据库吗？

不是。GDSQL 是一个数据管理插件，在 Godot 的 `ConfigFile` 系统之上提供类 SQL 接口。不使用 SQLite 或任何外部数据库引擎。所有数据存储在 `.cfg` 纯文本文件中。

### 需要数据库服务器吗？

不需要。GDSQL 完全自包含，无服务器进程、无网络连接、无外部依赖。

### 导出的游戏能用吗？

可以。同一个 API 在编辑器和导出游戏中都可用。GDSQL 是纯 GDScript 插件，没有编辑器专用依赖。

### 支持哪些 Godot 版本？

需要 **Godot 4.x**，不兼容 Godot 3.x。

## 数据与存储

### 数据存在哪里？

`.cfg` 文件中，每个数据库是一个目录，每个表是该目录下的一个 `.cfg` 文件。

### 能手动编辑 `.cfg` 文件吗？

可以。但要注意格式——无效语法会导致 GDSQL 加载时报错。

### 数据安全吗？

`.cfg` 是纯文本，有文件访问权限的人都能读取。敏感数据请使用 [内置加密](./encryption) 功能。

## SQL 与查询

### 支持所有 SQL 语法吗？

支持 SQL 的重要子集。详见 [SQL 引擎](./sql-engine) 页面。不支持 DDL 语句（CREATE TABLE 等）。

### 支持子查询吗？

支持。相关子查询和非相关子查询都可以。

## GBatis

### 必须使用 GBatis 吗？

不是。GBatis 是可选的。[链式 DAO API](./dao-api) 可以完成所有数据库操作。

### 链式 DAO 和 GBatis 能混用吗？

可以。在同一个项目中，简单查询用链式 DAO，复杂查询用 GBatis。

## 性能优化

- 使用 `set_indexed_props()` 创建内存索引
- 避免 `SELECT *`，只查需要的列
- 使用 `LIMIT` 减少结果集
- LRU 缓存自动加速重复查询

## 排错

### 编辑器中看不到 GDSQL 按钮
确认插件已启用：**项目 → 项目设置 → 插件 → GDSQL → 启用**。

### 查询返回空结果
检查数据库名、表名是否正确，表中是否有数据，WHERE 条件是否匹配。

### SQL "解析失败" 错误
检查 SQL 语法、表名和列名大小写是否匹配。
